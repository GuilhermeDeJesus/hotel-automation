# Histórico de Refatoração (Clean Architecture)

Registro curto das diretrizes mantidas após refactors.

## Princípios mantidos

1. Domínio desacoplado de frameworks.
2. Use cases dependem de abstrações.
3. Implementações concretas ficam na infraestrutura.
4. Camada de interface traduz protocolos externos para DTOs internos.

## Resultado

- Código mais testável.
- Menor acoplamento com provedores externos.
- Evolução mais previsível por camada.
